    üîπ Key Features:
  • Data is stored as documents (usually JSON or BSON format)
  • Each document is self-contained (can have different structures).
  • Schema-less or flexible schema.
  • Optimized for read/write performance and scalability.
  • Often used in horizontal scaling (easier to distribute across servers).
    ‚úÖ When to Use:
  • When your data is semi-structured or unstructured.
  • You need to store nested data (e.g., a user profile with addresses, settings, etc.).
  • You're building rapidly changing apps, like real-time analytics, CMSs, or mobile apps.
  • You prefer flexibility over strict structure.
